# 方案 B 归档：sherpa-onnx（备用）

## 1. 适用场景

1. 需要 Android/Windows 统一本地离线链路。
2. 弱网或无网场景占比高。
3. 对唤醒词、VAD、ASR、说话人能力希望同栈统一维护。

## 2. 接入范围（备用预案）

1. Android：KWS + VAD + ASR 本地运行。
2. Windows：会后加精与可选会中兜底。
3. 与会议模式状态机对齐（单录音链路复用）。

## 3. 优势

1. 离线能力强，网络依赖低。
2. 同栈跨平台，组件一致性好。
3. 唤醒词链路可与主录音链路融合实现。

## 4. 风险

1. 集成复杂度高于主线方案 A。
2. 模型体积、性能调优、设备兼容需要专项验证。
3. 首期交付周期会被拉长。

## 5. 启用条件（触发切换）

1. 主线方案 A 在弱网/离线指标连续不达标。
2. 会中可用性目标无法通过会后加精补偿。
3. 项目确认接受额外集成成本与周期。
