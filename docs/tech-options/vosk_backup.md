# 方案 C 归档：Vosk（备用）

## 1. 适用场景

1. 需要轻量级离线兜底能力。
2. 设备资源受限、部署成本敏感。
3. 首先追求“可用”而非“最佳精度”。

## 2. 接入范围（备用预案）

1. Android：离线应急 STT。
2. Windows：会后加精的最低保障链路。
3. 作为主线方案 A 的故障兜底，不替代主流程。

## 3. 优势

1. 部署轻，跨平台生态成熟。
2. 离线运行稳定，便于快速兜底。

## 4. 风险

1. 中文会议复杂场景精度通常低于主线与方案 B。
2. 说话人细粒度能力需要额外组件拼接。
3. 对多语言混说与重叠语音表现有限。

## 5. 启用条件（触发切换）

1. 主链路故障时的应急降级。
2. 指定低资源设备的特化部署。
3. 明确接受“可用优先、精度次之”的场景。
